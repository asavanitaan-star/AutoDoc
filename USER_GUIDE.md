# คู่มือการใช้งาน AutoDoc PCR

คู่มือนี้สำหรับ **Service Engineer** ที่ใช้แอปสร้างใบ Certificate of Calibration
ของเครื่อง Applied Biosystems Veriti™ Thermal Cycler

---

## 1. เข้าสู่ระบบครั้งแรก

### ถ้ามีบัญชีอยู่แล้ว
1. เปิดลิงก์แอป (เช่น `https://<โดเมนบริษัท>/`)
2. กรอก **ชื่อผู้ใช้** และ **รหัสผ่าน** → กด "เข้าสู่ระบบ"

### ถ้ายังไม่มีบัญชี — สมัครเอง
1. เปิดหน้า `/register.html` (มีลิงก์ "ยังไม่มีบัญชี? สมัครสมาชิก" จากหน้า login)
2. กรอกชื่อผู้ใช้ (ภาษาอังกฤษ/ตัวเลข), ชื่อที่แสดง, รหัสผ่าน (อย่างน้อย 6 ตัวอักษร)
3. กรอก **รหัสเชิญ** ที่ได้รับจากผู้ดูแลระบบ (ขอจากหัวหน้าทีม/แอดมิน)
4. กด "สมัครสมาชิก" — ระบบจะพาเข้าเว็บให้ทันทีโดยไม่ต้อง login ซ้ำ

> ถ้าไม่มีรหัสเชิญ หรือขึ้นข้อความ "ปิดการสมัครสมาชิกอยู่" ให้ติดต่อผู้ดูแลระบบ

---

## 2. สร้าง Certificate ใหม่

จากหน้า **รายการ** กด **"+ สร้างใบใหม่"** แล้วกรอกข้อมูลตามฟอร์ม *Planned
Maintenance Protocol (FM-SP-02-11)* ที่วัดได้จากเครื่องจริง:

### 2.1 ข้อมูลทั่วไป
| ช่องในแอป | กรอกจาก |
|---|---|
| Certificate No. | ระบบใส่เลขให้อัตโนมัติ (แก้เองได้ถ้าจำเป็น) |
| Calibration Date | วันที่ทำการ Calibrate |
| Organization Name | ชื่อหน่วยงาน/ลูกค้า |
| Organization Location | ที่อยู่หน่วยงาน |
| Instrument Model / Serial Number | รุ่นและ S/N ของเครื่อง |
| Calibration By | ชื่อวิศวกรผู้ทำ (ปกติดึงจากค่า default ในตั้งค่า) |

### 2.2 Performance
กรอกค่าที่วัดได้ 4 ค่า: **Ramp rate, Average cycle time, Cycle time SD,
Heated cover Verification** — ทุกช่องมีป้าย PASS/FAIL ขึ้นให้ทันทีตามเกณฑ์ spec

### 2.3 Temperature accuracy (85 °C / 45 °C)
กรอกค่าทั้ง 6 โซนของแต่ละ Set Point

### 2.4 Temperature Non Uniformity (TNU) — จุดสำคัญที่สุด
กรอกค่าจากฟอร์มวัดค่าให้ตรงกับช่องในแอปตามตารางนี้ **(ชื่อไม่ตรงกันตรงๆ
ต้องดูให้ดี)**:

| ในฟอร์มวัดค่า | กรอกในแอปช่อง | ไปออกในใบ Certificate เป็น |
|---|---|---|
| TNU: 95 °C (6 โซน + Overall) | **TNU 95 °C → DOWNRAMP** | DOWNRAMP TNU (95°C→60°C) |
| TNU: 60 °C (6 โซน + Overall) | **TNU 60 °C → UPRAMP** | UPRAMP TNU (60°C→95°C) |

### 2.5 บันทึก
- **บันทึก** — เซฟแล้วกลับไปหน้ารายการ
- **บันทึก & ดูใบเซอร์** — เซฟแล้วเปิดใบ Certificate ให้ดูทันที

ที่มุมล่างของฟอร์มจะมีป้าย **OVERALL: PASS/FAIL** สรุปผลรวมทุกค่า

---

## 3. ดู / พิมพ์ใบ Certificate

1. จากหน้ารายการ กด **"ดูใบเซอร์"** ที่แถวของ certificate นั้น
2. ตรวจสอบป้าย **ALL PASS** หรือ **พบค่าที่ FAIL** ที่แถบด้านบน
3. กด **🖨 พิมพ์ / บันทึกเป็น PDF**
4. ในหน้าต่างพิมพ์ของเบราว์เซอร์ ให้ตั้งค่า:
   - ขนาดกระดาษ: **A4**
   - Margin: **None**
   - แล้วเลือก "Save as PDF" หรือสั่งพิมพ์ได้เลย

ใบจะมี 2 หน้า: หน้าหลัก (Certificate of Calibration) และหน้า Reference
(อุปกรณ์ที่ใช้ calibrate + ลายเซ็นวิศวกร)

---

## 4. แก้ไข / ลบเอกสาร

จากหน้ารายการ แต่ละแถวมีปุ่ม:
- **แก้ไข** — เปิดฟอร์มเดิมเพื่อแก้ค่า
- **ลบ** — ลบเอกสารถาวร (มีกล่องยืนยันก่อนลบ)

---

## 5. ตั้งค่า (Settings)

หน้านี้กำหนดค่าที่ใช้ซ้ำในทุกใบ certificate:

| ส่วน | ใช้ทำอะไร |
|---|---|
| บริษัท / หัวเอกสาร | ชื่อบริษัท, ที่อยู่, ชื่อรุ่นสินค้าที่ขึ้นบนหัวใบเซอร์ |
| Service Engineer & ลายเซ็น | ชื่อ/ตำแหน่งวิศวกร + อัปโหลดรูปลายเซ็น (แสดงในหน้า Reference) |
| Certificate No. | ตั้ง prefix/เลขเริ่มต้น/รูปแบบเลขที่จะรันอัตโนมัติในใบถัดไป |
| Reference — Dedicated Equipment | ตารางอุปกรณ์ calibrate (Alpha technic, Reference probe ฯลฯ) |
| **บัญชีผู้ใช้** | เพิ่ม/ลบบัญชีวิศวกรคนอื่น |
| **การสมัครสมาชิก** | เปิด/ปิดหน้า `/register.html` และสุ่มรหัสเชิญใหม่ |
| เปลี่ยนรหัสผ่านของฉัน | เปลี่ยนรหัสผ่านบัญชีตัวเอง |
| Specification | เกณฑ์ Pass/Fail ของทุกค่า (แก้ได้ถ้ามาตรฐานเปลี่ยน) |

แก้เสร็จอย่าลืมกด **"บันทึกการตั้งค่า"** ที่ล่างสุดของหน้า

## 6. คำถามที่พบบ่อย

**Q: ลืมรหัสผ่าน ทำยังไง**
A: ตอนนี้ยังไม่มีระบบ "ลืมรหัสผ่าน" อัตโนมัติ — ให้ติดต่อเพื่อนร่วมทีมที่ยัง login
อยู่ ให้เข้าไปที่ตั้งค่า → บัญชีผู้ใช้ ลบบัญชีเดิมแล้วสมัครใหม่ หรือขอผู้ดูแลระบบ
ช่วยรีเซ็ตให้

**Q: ทำไมค่าบางช่องขึ้นสีแดง**
A: หมายถึงค่านั้นเกินเกณฑ์ spec ที่ตั้งไว้ (ดูหัวข้อ Specification) —
ตรวจสอบว่ากรอกค่าถูกต้องตรงกับฟอร์มวัดค่าหรือไม่

**Q: Cert No. ซ้ำกับใบเก่า ทำยังไง**
A: แก้เลขในช่อง "Certificate No." ตอนกรอกฟอร์มได้โดยตรง ระบบจะไม่บังคับให้ตรง
กับเลขที่รันอัตโนมัติเสมอไป

**Q: เข้าเว็บไม่ได้ ขึ้นหน้า login ตลอด**
A: Session หมดอายุ (30 วัน) หรือคุกกี้ถูกลบ — login ใหม่ได้เลย
